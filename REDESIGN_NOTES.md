# Maze Apple HIG Rehaul

## Visual Thesis
Calm, spacious, and content-first. This branch follows Apple-style hierarchy and material restraint without copying Apple branding.

## Main UX Bets
- Use softer translucency, bigger whitespace, and simpler page structure so the product feels easier to trust.
- Make the landing page more cinematic and less dashboard-like.
- Keep shared shells and auth flows consistent with the same glassy, high-clarity material language.

## Tradeoffs
- This branch invests more in atmosphere and first impression than in component-system rigor.
- It relies more on custom layout and CSS treatment, so it is visually distinct but less systemized than the shadcn branch.
